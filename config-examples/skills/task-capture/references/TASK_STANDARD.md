# Task Standard

This skill follows a lightweight GTD-style "next action" standard adapted for Markdown task lists.

## Why this format

Good todo items are easier to execute when they are:

- captured outside the head in a trusted system
- clarified into a concrete next action
- written as a visible verb plus object
- small enough to do without extra planning
- stored in one durable place

This is consistent with widely used task-management guidance from GTD and modern todo systems:

- GTD emphasizes capture, clarify, organize, and identifying the next physical action.
- Todoist's GTD guidance recommends turning vague items like `Taxes` into specific actions like `Discuss tax documents with Mary`.

## Local writing standard

Use this practical standard for each new task line:

1. One checkbox entry per task.
2. Start with an imperative verb in Polish.
3. Include the object or context needed to act.
4. Keep the task atomic when possible.
5. Add capture date as `➕ YYYY-MM-DD`.
6. Append to the end of the designated tasks file.

## Good examples

```markdown
- [ ] Kup chleb ➕ 2026-04-06
- [ ] Zadzwoń do dentysty i umów wizytę ➕ 2026-04-06
- [ ] Wyślij fakturę do Ani ➕ 2026-04-06
```

## Weak examples

```markdown
- [ ] Zakupy
- [ ] Dentysta
- [ ] Kupić chleb
```

Weak examples fail because they are vague or use infinitive phrasing instead of a direct next action.

## Scope limits

This skill is for quick task capture, not full project planning.

If the user describes a project rather than one action, capture the clearest next step or ask one short clarifying question.

## Research basis

The standard is based on publicly available guidance from:

- Getting Things Done (capture what has attention, clarify the next action, organize it in a trusted system)
- Todoist GTD guide (rewrite vague items into concrete actions)

Those sources point toward the same practical rule: a good task should tell the future-you exactly what to do.
