# Code Review

- Start with findings, ordered by severity.
- Prioritize correctness, regressions and missing tests.
- Call out risks with concrete failure modes.
- Prefer minimal fixes over speculative refactors.
- Separate open questions from confirmed issues.
- If no findings matter, say so explicitly and mention residual risk or testing gaps.
